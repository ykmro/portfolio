## 8. このアプリの懸念点とその対策
※最低1つは必ず書いてください。

このアプリが**うまくいかない可能性**と、
それに対して**どう向き合おうとしているか**を書いてください。

フラッシュカードの登録

### 懸念点①

- 何が問題になりそうか：
  - エンジニア向けの機能としてjsonを用いた一括アップロードを実装したいが、json自体はフォーマットのみ用意してフラッシュカードの内容はユーザーに生成AIで作ってもらう想定。個別登録の手間を省くための機能だが、そもそもプラットフォーム外で作業する必要がある機能は実装しないほうがいいのか
- なぜそう思うか：
  - カード登録がメインの操作なのに、そこでアプリ外に離脱させると離脱率が上がり継続利用されにくくなる
- そのために考えている対策：
  - あくまで「まとめて登録したい上級者向けのオプション」と位置づけて、メインの導線はアプリ内の手入力フォームにする

### 懸念点②

- 何が問題になりそうか：
  - ユーザーが有料教材の内容を転記していた場合、著作物の無断配布に該当する可能性がある
- なぜそう思うか：
  - 著作権侵害に該当する可能性があるため
- そのために考えている対策：
  - 利用規約を用意して注意喚起。通報機能を追加するなど
